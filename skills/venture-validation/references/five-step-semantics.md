# Five-Step Semantics

Use this reference when the user needs a cleaner understanding of what each step really means.

## The Five Problems

1. Demand
   - origin problem
   - who is the user, in what scene, with what pain

2. Solution
   - value problem
   - what is being offered, why it is better, and why the user should switch

3. Business Model
   - efficiency problem
   - whether the business can make money and continue making money

4. Growth
   - replication problem
   - how new users arrive repeatedly and whether growth can continue

5. Defensibility
   - endgame problem
   - what lets the business keep winning when competitors respond

## Why This Matters

Teams often mix these layers together:

- describing a user is not the same as proving demand
- building a feature is not the same as proving value
- getting revenue is not the same as proving healthy unit economics
- getting users once is not the same as proving repeatable growth
- having an idea is not the same as having a moat

This reference is useful when the team needs sharper language and better diagnosis.
